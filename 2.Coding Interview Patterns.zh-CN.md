## 1. 双指针模式简介

> 原文：https://bytebytego.com/courses/coding-patterns/two-pointers/introduction-to-two-pointers

## 直观理解

顾名思义，双指针模式是指一种利用两个指针的算法。但什么是指针呢？它是一个表示数据结构（如数组或链表）中索引或位置的变量。许多算法只使用单个指针来获取或跟踪单个元素：

![该图展示了数据结构（可能是数组或列表）中数据插入或修改的可视化描述。一个包含字母 'i' 的橙色小方块作为指示器或指针，位于方括号内的数字序列 `[ ... 14 5 5 20 ... ]` 上方。一个向下的橙色箭头从 'i' 指示器指向数字 '5'，表明 'i' 代表数据结构中的一个位置或索引，箭头表示在该特定索引处正在进行插入或修改。省略号 (...) 表示在序列的可见部分前后还有其他元素。整个图示说明了一个简单的数据操作，重点是在大型数据集的特定索引处插入或更新值。](https://bytebytego.com/images/courses/coding-patterns/two-pointers/introduction-to-two-pointers/image-01-00-1-OA7SGCBK.svg)

引入第二个指针开启了无限可能。最重要的是，我们现在可以进行**比较**。通过位于两个不同位置的指针，我们可以比较这些位置上的元素，并根据比较结果做出决策：

![该图展示了排序算法（可能是冒泡排序或插入排序等基于比较的排序）中比较步骤的可视化描述。图中显示了一个名为 `nums` 的数值数组，表示为 `[... 14 5 5 20 ...]`，这是大数组的部分视图。两个索引变量 `i`（橙色框）和 `j`（浅蓝色框）分别指向数组中的元素 14 和 5。箭头从 `i` 和 `j` 向下延伸以突出显示这些选定的元素。右侧的一个虚线框描述了函数调用 `compare(nums[i], nums[j])`，这意味着对索引 `i` 和 `j` 处的数值（本例中为 14 和 5）进行比较操作。指向该框右侧的箭头引出文本 “make decision”（做出决策），表明比较的结果（无论 `nums[i]` > `nums[j]`、`nums[i]` < `nums[j]` 还是 `nums[i]` == `nums[j]`）将决定排序算法的下一步。](https://bytebytego.com/images/courses/coding-patterns/two-pointers/introduction-to-two-pointers/image-01-00-2-Y7CSNC5F.svg)

在许多情况下，此类比较是使用两个嵌套的 for 循环完成的，这需要 $O(n^2)$ 的时间复杂度，其中 `n` 表示数据结构的长度。在下面的代码片段中，`i` 和 `j` 是用于比较数组中每两个元素的两个指针：

Python

JavaScript

Java

Cpp

```python
for i in range(n):
    for j in range(i + 1, n):
        compare(nums[i], nums[j])
```

通常，这种方法没有利用数据结构中可能存在的**可预测动态特性**（predictable dynamics）。具有可预测动态特性的数据结构的一个例子是已排序数组：当我们在已排序数组中移动指针时，我们可以预测移动到的值是变大还是变小。例如，在升序数组中向右移动指针，可以保证我们移动到的值大于或等于当前值：
![该图展示了有序数组中的编码模式或预测。顶部有一个包含字母 'i' 的橙色小方块向下指，表示索引或迭代器。下方显示了一个有序整数数组 `[1 2 3 4]`，下面有一条灰色线，标记为 'sorted array'（有序数组）。底部的一个虚线框包含一个以伪代码风格编写的预测语句：'prediction: if nums[i] == 2, then nums[i + 1] ≥ 2'。该语句表明，如果 `nums` 数组中索引 'i' 处的元素等于 2，那么下一个索引 (i+1) 处的元素必须大于或等于 2。来自 'i' 的箭头暗示 'i' 用于索引数组，这可能是算法或条件检查的一部分。](https://bytebytego.com/images/courses/coding-patterns/two-pointers/introduction-to-two-pointers/image-01-00-3-ASETSKFQ.svg)

正如你所看到的，具有可预测动态特性的数据结构允许我们以逻辑方式移动指针。利用这种可预测性可以优化时间和空间复杂度，我们将在本章中通过实际的面试题来说明这一点。

## 双指针策略

双指针算法通常通过消除嵌套 for 循环的需求，仅需 $O(n)$ 时间复杂度。使用双指针主要有三种策略。

**向内遍历**
这种方法让指针从数据结构的相对两端开始，并向中间靠拢：

![该图直观地描述了编码模式中的双指针方法，可能用于数组遍历或类似操作。图中显示了方括号内的一系列八个黑点，代表数据结构中的元素。在圆点上方，两个矩形框分别标记为 'left'（橙色）和 'right'（浅蓝色）。橙色虚线将 'left' 框连接到前四个点，表示指针从左向右遍历该部分。类似地，浅蓝色虚线将 'right' 框连接到最后四个点，显示指针从右向左移动。两个指针都汇聚在中心点，暗示算法中的交汇点。来自 'left' 和 'right' 框的箭头向下指，表示指针向中心移动的方向。整体结构说明了一种常见的算法策略，即两个指针从数据结构的相对两端开始，向彼此移动，直到它们相遇或满足特定条件。](https://bytebytego.com/images/courses/coding-patterns/two-pointers/introduction-to-two-pointers/image-01-00-4-SFTU7WGJ.svg)

指针向中心移动，根据比较结果调整位置，直到满足特定条件，或者它们相遇/交错。这对于需要比较数据结构不同端元素的题目非常理想。

**单向遍历**
在这种方法中，两个指针都从数据结构的同一端（通常是开头）开始，并向同一方向移动：
![该图展示了遍历数据结构（可能是数组或列表）的双指针法的视觉描述。结构显示为方括号 `[ ]` 内的一系列黑点，代表单个元素。一个标有“left”的橙色矩形用向下箭头指向点子集中的最左侧元素。带箭头的橙色虚线将该元素与其右侧的下几个元素连接起来，表示从左到右的遍历。类似地，一个标有“right”的浅蓝色矩形用向下箭头指向不同点子集中的最右侧元素。带箭头的浅蓝色虚线将该元素与其左侧的前几个元素连接起来，表示从右到左的遍历。由橙色和浅蓝色箭头指示的两个指针正向数据结构的中心移动，这表明了合并或比较过程，常用于归并排序或寻找具有特定总和的数对的双指针技术。](https://bytebytego.com/images/courses/coding-patterns/two-pointers/introduction-to-two-pointers/image-01-00-5-2KI3FUD6.svg)

这些指针通常用于两个不同但互补的目的。一个常见的应用是，我们让一个指针负责查找信息（通常是右指针），而另一个指针负责记录信息（通常是左指针）。

**阶段性遍历**
在这种方法中，我们先用一个指针进行遍历，当它指向满足特定条件的元素时，再用第二个指针进行遍历：

![该图展示了一个编码模式的图解，可能与数据处理或算法执行有关。图表显示了由一个大右箭头分隔的两个阶段。第一阶段描绘了方括号中一系列黑点，代表数据元素。一个标有“first”的橙色矩形框用箭头向下指向序列中的第二个元素，表示作用于该特定元素的进程或函数（“first”）。连接显示为数据序列上方形成弯曲圆弧的橙色虚线。第二阶段同样包含在方括号中，显示相同的数据序列，但具有不同的转换。一个标有“second”的浅蓝色矩形框用箭头向下指向这个新序列中的第二个元素，表示作用于同一元素的不同进程或函数（“second”）。连接显示为数据序列上方形成弯曲圆弧的浅蓝色虚线。在第二阶段上方显示了第三个相同的标有“first”的橙色框，表明“first”进程再次被应用，可能是迭代地或与“second”进程并行地。整体转换显示了初始数据序列如何通过“first”和“second”进程的顺序或并行应用而修改。](https://bytebytego.com/images/courses/coding-patterns/two-pointers/introduction-to-two-pointers/image-01-00-6-ZAL5SSYK.svg)

与单向遍历类似，两个指针各司其职。在这里，第一个指针用于搜索某些内容，一旦找到，第二个指针就会查找与第一个指针处的值相关的附加信息。

我们将在本章的所有题目中详细讨论这些技术。

## 何时使用双指针？

双指针算法通常需要线性数据结构，例如数组或链表。此外，当输入数据遵循某种可预测的动态规律（例如**有序数组**）时，通常暗示该问题可以使用双指针算法解决。

可预测的动态规律可以有多种形式。以回文字符串为例，它的对称模式允许我们逻辑上将两个指针向中心移动。随着你完成本章中的题目，你将学会更轻松地识别这些可预测的动态规律。
另一个可能表明问题可以使用双指针解决的迹象是，如果题目要求寻找一对**数值（pair of values）**，或者一个可以由两个数值生成的结果。

## 实际应用案例

**垃圾回收算法**：在内存压缩（Memory compaction）中——这是垃圾回收的关键部分——目标是通过消除已释放（即死亡）对象留下的间隙来释放连续的内存空间。双指针技术有助于高效地实现这一目标：“扫描（scan）”指针遍历堆以识别存活对象，而“空闲（free）”指针则跟踪存活对象应迁移到的下一个可用空间。随着“扫描”指针的移动，它会跳过死亡对象，并将存活对象移动到“空闲”指针指示的位置，通过将所有存活对象聚集在一起来压缩内存，从而释放出连续的内存块。

## 2. 两数之和 - 有序数组

> 原文：https://bytebytego.com/courses/coding-patterns/two-pointers/pair-sum-sorted

给定一个按升序排序的整数数组和一个目标值，返回数组中**和为目标值的任意一对数字的索引**。结果中索引的顺序不重要。如果没有找到匹配的对，则返回一个空数组。

#### 示例 1：

```python
输入: nums = [-5, -2, 3, 4, 6], target = 7
输出: [2, 3]
```

解释：`nums[2] + nums[3] = 3 + 4 = 7`

#### 示例 2：

```python
输入: nums = [1, 1, 1], target = 2
输出: [0, 1]
```

解释：其他有效的输出可以是 `[1, 0]`、`[0, 2]`、`[2, 0]`、`[1, 2]` 或 `[2, 1]`。

## 思路

解决此问题的暴力解法涉及检查所有可能的数对。这可以通过两个嵌套循环来完成：外层循环遍历数组以获取数对的第一个元素，内层循环遍历数组的其余部分以查找第二个元素。以下是该方法的代码片段：

Python

JavaScript

Java

```python
from typing import List

def pair_sum_sorted_brute_force(nums: List[int], target: int) -> List[int]:
    n = len(nums)
    for i in range(n):
        for j in range(i + 1, n):
            if nums[i] + nums[j] == target:
                return [i, j]
    return []
```

该方法的时间复杂度为 $O(n^2)$，其中 $n$ 表示数组的长度。这种方法没有考虑到输入数组是有序的。我们能否利用这一事实来提出更高效的解决方案？

在这里值得考虑**双指针法**，因为有序数组允许我们以逻辑化的方式移动指针。让我们通过下面的示例来看看它是如何工作的：

-   左指针：由于数组已排序，左指针指向的值始终小于或等于右指针指向的值。递增左指针将导致和大于或等于当前的和 1。

-   右指针：递减右指针将导致和小于或等于 1。

因此，我们应该递增左指针以寻找更大的和：

上文演示了使用向内遍历的双指针算法。让我们总结一下这个逻辑。对于 `left` 和 `right` 处的任何一对值：

-   如果它们的总和小于目标值，则递增 `left`，旨在使总和向目标值增加。
-   如果它们的总和大于目标值，则递减 `right`，旨在使总和向目标值减小。
-   如果它们的总和等于目标值，则返回 `[left, right]`。

当 `left` 和 `right` 指针相遇时，我们可以停止移动，因为这表明没有找到总和等于目标值的元素对。

```python
from typing import List
       
def pair_sum_sorted(nums: List[int], target: int) -> List[int]:
    left, right = 0, len(nums) - 1
    while left < right:
        sum = nums[left] + nums[right]
        # 如果总和较小，则递增左指针，旨在使总和向目标值增加。
        if sum < target:
            left += 1
        # 如果总和较大，则递减右指针，旨在使总和向目标值减小。
        elif sum > target:
            right -= 1
        # 如果找到了目标元素对，返回其索引。
        else:
            return [left, right]
    return []
```

## 3. 三数之和 (Triplet Sum)

> 数据源：https://bytebytego.com/courses/coding-patterns/two-pointers/triplet-sum

给定一个整数数组，返回所有满足 `a + b + c = 0` 的三元组 `[a, b, c]`。解集不得包含重复的三元组（例如，`[1, 2, 3]` 和 `[2, 3, 1]` 被视为重复）。如果没有找到这样的三元组，则返回一个空数组。

每个三元组内的元素可以按任意顺序排列，输出结果也可以按任意顺序返回。

#### 示例：

```python
输入: nums = [0, -1, 2, -3, 1]
输出: [[-3, 1, 2], [-1, 0, 1]]
```

## 直觉 (Intuition)

暴力解法涉及检查数组中所有可能的三元组，看它们的和是否为零。这可以通过三个嵌套循环来实现，遍历三个元素的每种组合。

通过对每个三元组进行排序，可以避免重复的三元组，这确保了具有不同表示形式的相同三元组（例如 `[1, 3, 2]` 和 `[3, 2, 1]`）具有一致的顺序（例如 `[1, 2, 3]`）。排序后，我们可以将这些三元组添加到哈希集合（hash set）中。这样，如果再次遇到相同的三元组，哈希集合将只保留一个实例。以下是该方法的代码片段：

Python

JavaScript

Java

```python
from typing import List

def triplet_sum_brute_force(nums: List[int]) -> List[List[int]]:
    n = len(nums)
    # 使用哈希集合确保不添加重复的三元组。
    triplets = set()
    # 遍历所有三元组的索引。
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    # 在放入哈希集合之前对三元组进行排序。
                    triplet = tuple(sorted([nums[i], nums[j], nums[k]]))
                    triplets.add(triplet)
    return [list(triplet) for triplet in triplets]
```

## 实现

根据上述思路，我们知道需要对 `pair_sum_sorted` 函数进行微调，以避免重复的三元组。我们还需要传入一个起始值（start value），以指示执行两数之和算法的子数组起点。除此之外，双指针逻辑与“两数之和 - 已排序数组”几乎完全相同。

Python

JavaScript

Java

```python
from typing import List

def triplet_sum(nums: List[int]) -> List[List[int]]:
    triplets = []
    nums.sort()
    for i in range(len(nums)):
        # 优化：仅由正数组成的三元组之和永远不会为 0。
        if nums[i] > 0:
            break
        # 为了避免重复的三元组，如果 'a' 与前一个数字相同，则跳过。
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        # 寻找所有和为目标值 '-a' ('-nums[i]') 的数对。
        pairs = pair_sum_sorted_all_pairs(nums, i + 1, -nums[i])
        for pair in pairs:
            triplets.append([nums[i]] + pair)
    return triplets

def pair_sum_sorted_all_pairs(nums: List[int], start: int, target: int) -> List[int]:
    pairs = []
    left, right = start, len(nums) - 1
    while left < right:
        sum = nums[left] + nums[right]
        if sum == target:
            pairs.append([nums[left], nums[right]])
            left += 1 # 为了避免重复的 '[b, c]' 数对，如果 'b' 与前一个数字相同，则跳过。
            while left < right and nums[left] == nums[left - 1]:
                left += 1
        elif sum < target:
            left += 1
        else:
            right -= 1
    return pairs
```

### 复杂度分析

**时间复杂度：** `triplet_sum` 的时间复杂度为 O(n2)O(n^2)O(n2)。原因如下：

-   我们首先对数组进行排序，这需要 O(nlog⁡(n))O(n\\log(n))O(nlog(n)) 的时间。
-   然后，对于数组中的 nnn 个元素，我们最多调用一次 `pair_sum_sorted_all_pairs`，其运行时间为 O(n)O(n)O(n)。

因此，总的时间复杂度为 O(log⁡(n))+O(n2)\=O(n2)O(\\log(n))+O(n^2)=O(n^2)O(log(n))+O(n2)\=O(n2)。

**空间复杂度：** 空间复杂度为 O(n)O(n)O(n)，这是由于 Python 排序算法所占用的空间。需要注意的是，此复杂度不包括输出数组 `triplets`，因为我们只关注算法使用的额外空间，而不是输出本身所需的空间。

如果面试官问及包含输出数组在内的空间复杂度，那么它将是 O(n2)O(n^2)O(n2)。这是因为在最坏情况下，`pair_sum_sorted_all_pairs` 函数可能会向输出中添加大约 nnn 个数对。由于该函数被调用了大约 nnn 次，因此总的空间复杂度为 O(n2)O(n^2)O(n2)。

-   如果 `left` 和 `right` 指向的字母数字字符相同，则将两个指针同时向内移动，以处理下一对字符。

-   如果不同，则该字符串不是回文：返回 false。

如果我们成功比较了所有字符对而没有返回 false，则该字符串是回文，我们应该返回 true。

**处理非字母数字字符**
现在，让我们探索如何处理包含非字母数字字符的回文检查。

由于非字母数字字符不影响字符串是否为回文，我们应该跳过它们。这可以通过以下方法实现，确保调整左右指针以仅关注字母数字字符：

-   递增 `left`，直到它指向的字符是字母数字。

-   递减 `right`，直到它指向的字符是字母数字。

## 实现

在 Python 中，我们可以使用内置的 `isalnum` 方法来检查一个字符是否为字母数字。

Python

JavaScript
Java

```python
def is_palindrome_valid(s: str) -> bool:
    left, right = 0, len(s) - 1
    while left < right:
        # 从左侧跳过非字母数字字符。
        while left < right and not s[left].isalnum():
            left += 1
        # 从右侧跳过非字母数字字符。
        while left < right and not s[right].isalnum():
            right -= 1
        # 如果左右指针指向的字符不匹配，则该字符串不是回文。
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    return True
```

### 复杂度分析

**时间复杂度：** `is_palindrome_valid` 的时间复杂度为 $O(n)$，其中 $n$ 表示字符串的长度。这是因为我们使用双指针技术执行了大约 $n$ 次迭代。

**空间复杂度：** 我们只分配了常数个变量，因此空间复杂度为 $O(1)$。

## 直觉

如果我们有两条垂直线 `heights[i]` 和 `heights[j]`，这两条线之间可以容纳的水量为 `min(heights[i], heights[j]) * (j - i)`，其中 `j - i` 代表容器的宽度。我们取最小高度，因为注水超过这个高度会导致溢出。

![图片展示了一个计算过程及其公式和结果的图形说明。左侧显示了一个二维笛卡尔坐标系，其中一个浅蓝色矩形代表水量。矩形的底边在 x 轴上从 x=0 延伸到 x=2，高度沿 y 轴为 3 个单位，从 y=0 到 y=3。矩形内还有一个位于 x=1 的垂直线段。一个灰色的弯曲箭头从矩形的右上角指向右侧，表示计算结果的方向。在图形右侧，显示了公式 'water = min(4, 3) ⋅ (2 - 0)'，用于计算水量。'min(4, 3)' 找出 4 和 3 之间的最小值（即 3），然后将其乘以底边长度 (2 - 0 = 2)。随后的行显示了计算步骤：'3 ⋅ 2' 以及最终结果 '6'，代表总水量（可能以任意单位表示）。](https://bytebytego.com/images/courses/coding-patterns/two-pointers/largest-container/image-01-04-1-T7WVY3OS.svg)

换句话说，容器的面积取决于两点：

-   矩形的**宽度**。
-   矩形的**高度**，由两条线中较短的那条决定。

解决此问题的暴力解法涉及检查所有线条对，并返回每对线条之间发现的最大面积：

Python

JavaScript

Java

```python
from typing import List
    
def largest_container_brute_force(heights: List[int]) -> int:
   n = len(heights)
   max_water = 0
   # 找到所有线条对之间存储的最大水量。
   for i in range(n):
       for j in range(i + 1, n):
           water = min(heights[i], heights[j]) * (j - i)
           max_water = max(max_water, water)
   return max_water
```

搜索所有可能的数值对需要 $O(n^2)$ 时间，其中 $n$ 表示数组的长度。让我们寻找一种更高效的解决方案。

为了获得最大的容器，我们希望高度和宽度都尽可能大。
寻找具有最大高度的容器的方法并不直观，因为数组中线条的高度并没有明显的规律。然而，我们确实知道宽度最大的容器：即起始于索引 `0` 且终止于索引 `n - 1` 的容器。

因此，我们可以从最大化宽度开始，在数组的两端各设置一个指针。然后，我们可以通过将这两个指针向内移动来逐渐减小宽度，希望能找到一个高度更大的容器，从而可能产生更大的面积。这表明我们可以使用**双指针 (two-pointer)** 模式来解决这个问题。

> 将指针向内移动意味着将左指针向右移，或将右指针向左移，从而有效地缩小它们之间的间距。

```python
from typing import List

def largest_container(heights: List[int]) -> int:
    max_water = 0
    left, right = 0, len(heights) - 1
    while (left < right):
        # 计算当前两线之间包含的水量。
        water = min(heights[left], heights[right]) * (right - left)
        max_water = max(max_water, water)
        # 向内移动指针，始终移动较短线条处的指针。
        # 如果两条线高度相同，则将两个指针都向内移动。
        if (heights[left] < heights[right]):
            left += 1
        elif (heights[left] > heights[right]):
            right -= 1
        else:
            left += 1
            right -= 1
    return max_water
```

## 6. 移动零至末尾

> 原文：https://bytebytego.com/courses/coding-patterns/two-pointers/shift-zeros-to-the-end

给定一个整数数组，原地修改该数组，将**所有零移动到末尾**，同时保持非零元素的相对顺序。

#### 示例：

```python
输入: nums = [0, 1, 0, 3, 2]
输出: [1, 3, 2, 0, 0]
```

## 直觉

这个问题有三个主要要求：

1.  将所有零移动到数组末尾。
2.  保持非零元素的相对顺序。
3.  原地进行修改。

解决此问题的一种朴素方法是使用一个辅助数组 (`temp`) 来构建输出。我们可以将 `nums` 中所有的非零元素从左侧开始添加到这个临时数组中，其余部分保留为零。然后，只需将输入数组设置为等于 `temp`。

> 通过首先识别并从数组左侧移动非零元素，我们可以确保在将它们添加到输出时保留其顺序：

Python

JavaScript

Java

```python
def shift_zeros_to_the_end_naive(nums: List[int]) -> None:
    temp = [0] * len(nums)
    i = 0
    # 将所有非零元素添加到 'temp' 的左侧。
    for num in nums:
        if num != 0:
            temp[i] = num
            i += 1
    # 将 'nums' 设置为 'temp'。
    for j in range(len(nums)):
        nums[j] = temp[j]
```

不幸的是，这个解决方案违反了原地修改输入数组的第三个要求。

然而，这种方法仍然提供了有价值的见解。特别是，请注意该解决方案关注的是非零元素而不是零。这意味着如果我们把目标改为**将所有非零元素移动到数组的左侧**，那么零自然就会落在右侧。因此，我们只需要关注非零元素：

## 实现

你可能已经注意到，无论右指针指向的是零还是非零，我们总是向前移动它。这使得我们可以使用 for 循环来迭代右指针。

Python

JavaScript

Java

```python
def shift_zeros_to_the_end(nums: List[int]) -> None:
    # 'left' 指针用于定位非零元素。
    left = 0
    # 使用 'right' 指针遍历数组以定位非零元素。
    for right in range(len(nums)):
        if nums[right] != 0:
            if right != left:
                nums[left], nums[right] = nums[right], nums[left]
            # 递增 'left'，因为它现在指向一个已被非零元素占据的位置。
            left += 1
```

### 复杂度分析

**时间复杂度：** `shift_zeros_to_the_end` 的时间复杂度为 O(n)，其中 n 表示数组的长度。这是因为我们对输入数组进行了一次遍历。

**空间复杂度：** 空间复杂度为 O(1)，因为移位操作是原地（in place）完成的。

## 7. 下一个字典序序列

> 原文：https://bytebytego.com/courses/coding-patterns/two-pointers/next-lexicographical-sequence

给定一个由小写英文字母组成的字符串，重新排列字符以形成一个新字符串，该字符串表示**字典序**（字母顺序）中的**下一个紧邻序列**。如果给定的字符串在所有可能的排列中已经是字典序最后一个，则返回字典序第一个的排列。

#### 示例 1：

```python
输入: s = 'abcd'
输出: 'abdc'
```

解释：在重新排列 `"abcd"` 后，`"abdc"` 是字典序中的下一个序列。

#### 示例 2：

```python
输入: s = 'dcba'
输出: 'abcd'
```

解释：由于 `"dcba"` 是字典序中的最后一个序列，我们返回第一个序列：`"abcd"`。

#### 约束条件：

-   字符串至少包含一个字符。

在确定了要重新排列的最短后缀后，下一个目标是重新排列该后缀，以实现**尽可能小的增量**。我们必须从轴点开始重新排列，因为后缀的其余部分已经处于其最大的排列状态。

为了使轴点位置的字符变大，我们需要**将轴点与右侧比它大的字符进行交换**。

在我们的示例中，我们的轴点（‘c’）应该与哪个字符交换？我们希望将其与比 ‘c’ 大的字符交换，但不能比它大*太多*，因为增量应该尽可能小。让我们看看如何找到这样一个字符。

就这样，我们找到了下一个字典序序列！在这个问题中使用的双指针策略是阶段性遍历，即我们首先确定枢轴（pivot），然后确定相对于它的最右侧后继（rightmost successor）。

确定下一个字典序序列涉及多个步骤。以下是总结：

1.  **定位枢轴。**

-   枢轴是从字符串右侧开始，第一个打破非递增序列的字符。
-   如果找不到枢轴，说明字符串已经处于其最后一个字典序序列，结果只需将字符串反转即可。

2.  **找到枢轴的最右侧后继。**

3.  **将最右侧后继与枢轴交换**，以增加后缀的字典序。

4.  **反转枢轴之后的后缀**，以使其排列最小。

```python
def next_lexicographical_sequence(s: str) -> str:
    letters = list(s)
    # 定位枢轴，即从右侧开始第一个打破非递增顺序的字符。
    # 从倒数第二个位置开始搜索。
    pivot = len(letters) - 2
    while pivot >= 0 and letters[pivot] >= letters[pivot + 1]:
        pivot -= 1
    # 如果找不到枢轴，说明字符串已经是最大的排列。
    # 在这种情况下，反转字符串以获得最小的排列。
    if pivot == -1:
        return ''.join(reversed(letters))
    # 找到枢轴的最右侧后继。
    rightmost_successor = len(letters) - 1
    while letters[rightmost_successor] <= letters[pivot]:
        rightmost_successor -= 1
    # 将最右侧后继与枢轴交换，以增加后缀的字典序。
    letters[pivot], letters[rightmost_successor] = (letters[rightmost_successor], letters[pivot])
    # 反转枢轴之后的后缀，以使其排列最小。
    letters[pivot + 1:] = reversed(letters[pivot + 1:])
    return ''.join(letters)
```

### 复杂度分析

**时间复杂度：** `next_lexicographical_sequence` 的时间复杂度为 $O(n)$，其中 $n$ 表示输入字符串的长度。这是因为我们对字符串进行了最多两次迭代：一次用于寻找枢轴，另一次用于在后缀中寻找值大于枢轴的最右侧字符。我们还执行了一次反转操作，耗时 $O(n)$。

**空间复杂度：** 空间复杂度为 $O(n)$，这是由于 `letters` 列表占用的空间。在 Python 中，需要这种额外空间是因为字符串是不可变的，因此必须将输入字符串存储为列表。
